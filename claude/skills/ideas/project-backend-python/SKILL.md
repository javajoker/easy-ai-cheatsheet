# Project Backend Generator (Python)

Step 5b (parallel with project-frontend) of the project-quick-start workflow.
**Position: AFTER task-breakdown.**

Builds a production-grade Python backend from THREE sources of truth:
1. Project docs (PRD, tech design) — the *what*
2. Task breakdown (AGENT.md, DEPENDENCY_GRAPH.md, task files) — the *how-organised*
3. User overrides if provided — explicit decisions

The generated code's package organisation mirrors the task breakdown's backend
components, so each backend task ID (e.g. `BE-003 Auth Service`) maps to a real
Python sub-package under `app/modules/`. When an AI agent picks up a task, the
corresponding package already exists.

This is **not** scaffolding — every endpoint specified in the tech design is fully
implemented with validation, error handling, auth checks, and database operations.

## Step 2 — Tech Stack (defaults)

Unless AGENT.md / tech design specified otherwise:

- **Language**: Python 3.12+
- **Web framework**: FastAPI 0.110+ (fallback: Django REST framework, Flask)
- **ASGI server**: uvicorn (production: gunicorn + uvicorn workers)
- **ORM**: SQLAlchemy 2.0+ (async) (fallback: Tortoise ORM, sync SQLAlchemy)
- **Database**: PostgreSQL 15 + asyncpg driver
- **Migrations**: Alembic
- **Cache**: Redis 7 via redis-py / aioredis
- **Job queue**: Celery (mature, full-featured) or arq (async-native, lighter)
- **Validation / serialisation**: Pydantic v2 (built into FastAPI)
- **Auth**: python-jose (RS256) + passlib[bcrypt]
- **i18n**: Babel + custom middleware (FastAPI doesn't ship i18n)
- **Logger**: structlog (structured) + loguru (DX)
- **Config**: pydantic-settings
- **Testing**: pytest + pytest-asyncio + httpx + testcontainers
- **Email**: SendGrid Python SDK (sendgrid)
- **File storage**: boto3 (aws-sdk for S3)
- **Monitoring**: sentry-sdk[fastapi]
- **Package management**: Poetry or uv (uv preferred for speed)
- **Container**: Docker + docker-compose
- **OpenAPI**: auto-generated by FastAPI from Pydantic models — no extra setup

If tech design or AGENT.md specifies different choices, follow them.

### Key alignment rule
For every backend task in the task breakdown:
1. Create a Python package in `app/modules/` — name is task ID lowercased with
   hyphens replaced by underscores (e.g. `BE-003 Auth Service` → `be003_auth`)
2. Add a `README.md` linking back to the source task file
3. Place all module-scoped code (routes, service, schemas, models, repository,
   tasks, tests) inside
4. Each module exports an `APIRouter` from `routes.py`; `app/main.py` registers
   them all in dependency order from the task graph

This gives a 1-to-1 mapping: when an AI agent picks up `BE-003 Auth Service`, it
knows exactly where to work.

## Step 5 — i18n Setup (Babel)

FastAPI doesn't ship with i18n, so we use Babel + a custom middleware.

### Configuration

```python
# app/core/i18n.py
from babel.support import Translations
from pathlib import Path

LOCALES_DIR = Path(__file__).parent.parent.parent / "locales"
SUPPORTED = ["en", "zh_TW"]
DEFAULT = "en"

_cache: dict[str, Translations] = {}

def get_translations(lang: str) -> Translations:
    """Return Babel Translations for the requested language with English fallback."""
    if lang not in SUPPORTED:
        lang = DEFAULT
    if lang not in _cache:
        _cache[lang] = Translations.load(LOCALES_DIR, [lang], domain="messages")
    return _cache[lang]

def translate(key: str, lang: str, **kwargs) -> str:
    t = get_translations(lang)
    msg = t.gettext(key)
    return msg.format(**kwargs) if kwargs else msg
```

### Per-request localiser middleware

```python
# app/core/middleware.py
from fastapi import Request
from starlette.middleware.base import BaseHTTPMiddleware

class I18nMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        accept = request.headers.get("accept-language", "en")
        lang = pick_language(accept)  # parse Accept-Language header
        request.state.lang = lang
        response = await call_next(request)
        return response

def get_lang(request: Request) -> str:
    return getattr(request.state, "lang", "en")
```

### Localised errors

```python
# app/core/errors.py
class APIError(Exception):
    def __init__(self, code: str, status: int, msg_id: str, **details):
        self.code = code
        self.status = status
        self.msg_id = msg_id
        self.details = details

# Global exception handler in app/main.py
@app.exception_handler(APIError)
async def api_error_handler(request: Request, exc: APIError):
    lang = get_lang(request)
    msg = translate(exc.msg_id, lang)
    return JSONResponse(
        status_code=exc.status,
        content={"error": {"code": exc.code, "message": msg, **exc.details}},
    )

# Usage in routes:
raise APIError("AUTH_INVALID_CREDENTIALS", 401, "auth.invalid_credentials")
```

## Step 6 — Auth Implementation

### JWT with RS256

```python
# app/core/jwt.py
from datetime import datetime, timedelta, timezone
from jose import jwt, JWTError
from uuid import UUID

from app.core.config import settings

ALGO = "RS256"

def sign_access_token(user_id: UUID, roles: list[str]) -> str:
    payload = {
        "sub": str(user_id),
        "roles": roles,
        "exp": datetime.now(tz=timezone.utc) + timedelta(minutes=15),
    }
    return jwt.encode(payload, settings.JWT_PRIVATE_KEY, algorithm=ALGO)

def sign_refresh_token(user_id: UUID) -> str:
    payload = {
        "sub": str(user_id),
        "type": "refresh",
        "exp": datetime.now(tz=timezone.utc) + timedelta(days=30),
    }
    return jwt.encode(payload, settings.JWT_PRIVATE_KEY, algorithm=ALGO)

def verify_token(token: str) -> dict:
    return jwt.decode(token, settings.JWT_PUBLIC_KEY, algorithms=[ALGO])
```

### Auth dependency (FastAPI Depends)

```python
# app/core/deps.py
from fastapi import Depends, Header
from app.core.errors import APIError
from app.core.jwt import verify_token

async def require_auth(authorization: str = Header(...)) -> dict:
    if not authorization.startswith("Bearer "):
        raise APIError("AUTH_MISSING_TOKEN", 401, "auth.missing_token")
    token = authorization[7:]
    try:
        return verify_token(token)
    except JWTError:
        raise APIError("AUTH_INVALID_TOKEN", 401, "auth.invalid_token")

async def require_role(*allowed: str):
    async def checker(claims: dict = Depends(require_auth)) -> dict:
        if not any(r in claims["roles"] for r in allowed):
            raise APIError("AUTH_FORBIDDEN", 403, "auth.forbidden")
        return claims
    return checker
```

### Refresh token rotation
- Refresh tokens stored in Redis with TTL = 30 days
- On refresh: validate, issue new pair, blocklist old refresh token in Redis
- On logout: blocklist refresh token

## Step 7 — Endpoint Implementation

For EVERY endpoint in TECH_DESIGN.md section 4:

1. **Find which task module owns this endpoint**
2. **Place route inside that module's `routes.py`**
3. **Pydantic schemas** in module's `schemas.py`:
   ```python
   from pydantic import BaseModel, Field, EmailStr

   class CreateIPAssetRequest(BaseModel):
       title: str = Field(..., min_length=1, max_length=200)
       description: str = Field("", max_length=2000)
       category: str = Field(..., pattern="^(art|music|video)$")

   class IPAssetResponse(BaseModel):
       id: UUID
       title: str
       description: str
       category: str
       created_at: datetime

       class Config:
           from_attributes = True
   ```
4. **Route handler** in module's `routes.py`:
   ```python
   from fastapi import APIRouter, Depends

   router = APIRouter(prefix="/ip-assets", tags=["IP Assets"])

   @router.post("", response_model=IPAssetResponse, status_code=201)
   async def create(
       body: CreateIPAssetRequest,
       claims: dict = Depends(require_auth),
       svc: IPAssetService = Depends(get_ip_asset_service),
   ):
       return await svc.create(user_id=UUID(claims["sub"]), data=body)
   ```
5. **Service function** in module's `service.py`:
   - Business logic
   - Calls repository for DB operations
   - Authorisation checks (ownership, etc.)
   - Enqueues background tasks via Celery/arq
6. **Repository** in module's `repository.py`:
   - SQLAlchemy queries only, no business logic
7. **Tests** in module's `tests/`:
   - Happy path with httpx AsyncClient
   - Auth failure
   - Authorisation failure
   - Validation failure
   - Not found

### Module registration

Each module exports its router:

```python
# app/modules/be003_auth/__init__.py
from .routes import router
__all__ = ["router"]
```

`app/main.py` registers them in dependency order from the task graph:

```python

app = FastAPI(title="My API", version="1.0.0")

app.include_router(be003_auth.router, prefix="/api/v1")
app.include_router(be007_ip_assets.router, prefix="/api/v1")
app.include_router(be012_payments.router, prefix="/api/v1")
```

### Endpoint coverage checklist
Every endpoint in TECH_DESIGN must:
- [ ] Belong to a task module (per task breakdown)
- [ ] Have a route in module's `routes.py`
- [ ] Have request + response Pydantic schemas in `schemas.py`
- [ ] Have a service function (no business logic in routes)
- [ ] Have an integration test in module's `tests/`
- [ ] Be included in `app/main.py` router registration
- [ ] Appear in auto-generated OpenAPI at `/docs`

NEVER skip an endpoint. NEVER stub one with `# TODO`.

## Step 8 — Background Workers (Celery or arq)

Choose Celery for full-featured legacy ecosystem, arq for async-native lighter weight.
Default to Celery unless the user prefers async-native.

For each job in TECH_DESIGN section 7:

1. Identify which task module owns the job
2. Place task in that module's `tasks.py`
3. Producer (service that enqueues) is in same module's `service.py`

```python
# app/modules/be008_email/tasks.py
from app.core.queue import celery_app
from app.integrations.email.sendgrid import send_email

@celery_app.task(name="email.send", bind=True, max_retries=3)
def send_email_task(self, to: str, template: str, data: dict):
    try:
        send_email(to=to, template=template, data=data)
    except Exception as exc:
        raise self.retry(exc=exc, countdown=2 ** self.request.retries)

# Enqueueing from a service:
# send_email_task.apply_async(kwargs={"to": user.email, "template": "welcome", "data": {...}})
```

Workers run as separate process via `celery -A app.worker worker --loglevel=info`.

## Step 12 — Generation Strategy for Large Backends

If the tech design has > 30 endpoints, generate in batches aligned with task IDs:

**Batch 1 (foundation)**: project layout, pyproject.toml, app/core (config, db,
  redis, jwt, i18n, logger, middleware, errors, queue, deps), Makefile, Dockerfile,
  Alembic init
**Batch 2 (auth + user modules)**: tasks with no dependencies or auth-only
**Batch 3 (core domain modules)**: tasks depending on auth
**Batch 4 (integrations + admin modules)**: tasks depending on multiple core modules
